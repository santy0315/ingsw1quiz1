Que es?

Es una combinación entre el modelo waterfall y un modelo de iteraciones, la cual consiste en seguir ciclos crecientes de cuatro fases cada uno, que se van realizando siguiendo una forma de espiral. se pasa por cada ciclo con fases bien definidas, las cuales se hacen en forma de espiral. Sus fases son: 1. Comunicación con el cliente 2. Planificación 3. Análisis de riesgos 4. Ingeniería 5. Evaluación del cliente 6. Construcción y entrega.

Tomado de: ¿Qué es el desarrollo en Espiral?

Ventajas y desventajas de esta metodología

Ventajas

Entre sus pricipales ventajas destacan:

  • Se pueden realizar cambios en cualquiera de las etapas
  • Se utilizan en proyectos grandes que esten sujetos a riesgos

Desventajas

Entre sus pricipales desventajas destacan:

  • Se genera mas documentacion al tener fases intermedias
  • No es aconsejable para proyectos pequeños, el coste no es rentable

Conclusiones

Para concluir esto modelo es importante ya que es utilizado para la realización de proyectos muy grandes, ya que este utiliza prototipos para la medición de riesgos, los cual hace que sea muy efectivo, siempre y cuando como en la mayoría de metodologías se siga de manera correcta cada una de sus fases; también consideró que al realizarse un análisis de riego se le puede dar a los proyectos un análisis realista para su correcta elaboración.
